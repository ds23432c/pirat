# ТО «ПИРАТЫ» — бронирование столов

Сайт бронирования столов для творческого объединения «Пираты»: схема зала
как на плане, бронь места в один клик, модальное окно с проверкой телефона
`+7`, и админ-панель с подтверждением заявок и редактором схемы.

Стек: **Node.js + Express + MySQL**. Развёртывание: **Railway**.

---


## Как работает бронь

1. Гость нажимает на **свободный** стол → модальное окно (имя, фамилия,
   телефон с маской и проверкой `+7`).
2. После отправки стол становится **серым** (`pending`) — бронь ждёт
   подтверждения. Повторно его уже не забронировать.
3. Администратор в панели нажимает **«Подтвердить»** → стол становится
   **красным** (`confirmed`). Сайт у гостей сам обновит цвет (опрос каждые 12 с).
4. Если админ нажимает **«Отменить»** — стол снова свободен.

## Админ-панель

Адрес: `https://ваш-домен/admin.html`
Пароль задаётся переменной `ADMIN_PASSWORD` (по умолчанию `piraty2019` —
**обязательно смените**).

Возможности:
- **Заявки** — список всех броней, подтверждение / отмена / удаление, фильтр.
- **Концерт** — название, дата, место, описание (показываются на главной).
- **Столы и схема** — перетаскивание столов мышью, изменение вместимости,
  цены, формы (круг/квадрат), включение/выключение, добавление и удаление.

## Запуск локально

```bash
npm install
cp .env.example .env      # впишите данные своей MySQL
npm start                 # http://localhost:3000
```

Таблицы и стартовая расстановка 13 столов создаются автоматически при первом
запуске.

## Деплой на Railway (по шагам)

1. Создайте проект на [railway.app](https://railway.app) → **New Project**.
2. **Add MySQL**: в проекте → `+ New` → `Database` → `Add MySQL`.
   Railway сам создаст переменную `MYSQL_URL`.
3. Добавьте сервис приложения: `+ New` → `GitHub Repo` (залейте эту папку в
   репозиторий) **или** `Empty Service` + Railway CLI (`railway up`).
4. В сервисе приложения → **Variables** добавьте:
   - `MYSQL_URL` → значение из MySQL-сервиса
     (вкладка MySQL → Variables → скопируйте `MYSQL_URL`),
   - `ADMIN_PASSWORD` → ваш пароль администратора.
   > `PORT` Railway подставляет сам — задавать не нужно.
5. Дождитесь деплоя, откройте сгенерированный домен (Settings → Networking →
   Generate Domain).

Готово. Главная — на корне домена, админка — на `/admin.html`.

## Чтобы сайт «засыпал» без посетителей и быстро просыпался

На Railway это встроенная функция **Serverless** (бывш. App Sleeping).
Включается в **один тумблер**, без кода:

**Сервис приложения → Settings → Deploy → Serverless → Enable Serverless.**
После включения сделайте redeploy (Cmd/Ctrl + K → «Deploy latest commit»).

Как это работает:
- Если ~**10 минут** нет активности — сервис останавливается (платить за
  простой не нужно).
- Первый же запрос (заход на сайт) его **будит**. Пересборки нет — Railway
  поднимает готовый деплой, поэтому старт быстрый, обычно несколько секунд
  («холодный старт»). Иногда самый первый запрос может вернуть `502` — просто
  обновите страницу.

> Порог именно ~10 минут (Railway не даёт выставить 3–4 часа), но для экономии
> это даже лучше — сервис засыпает раньше.

Важные нюансы под эту схему (уже учтено в коде):
- Пул соединений к MySQL **не держит** постоянный keep-alive и закрывает
  простаивающие соединения через 60 с — иначе постоянный трафик к БД не дал бы
  сервису уснуть.
- **Serverless включайте только на сервисе приложения**, а на самом
  MySQL-сервисе — **не включайте** (база должна быть доступна).

Совет по бюджету: Workspace → Usage → Set Usage Limits — поставьте Hard limit
(мин. $10), чтобы расходы были предсказуемыми.
